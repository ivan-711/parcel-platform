# Cash Flow Breakdown Visualization — Research & Recommendation

> Date: 2026-04-05
> Status: Recommendation — replace waterfall chart with proportional breakdown list

---

## 1. The Problem with the Waterfall

The current `WaterfallChart.tsx` uses a Recharts stacked BarChart with invisible spacer bars to create a waterfall effect. The question it answers is correct — "Here's your $1,830 rent. Here's what gets deducted. Here's what you keep." — but the execution has issues:

- **Floating bars confuse users.** UX research on waterfall charts consistently shows high comprehension failure rates. Users ask "why are the bars floating?" and "what do the different bars represent?" (Storytelling With Data, Domo).
- **No RE tool uses waterfall charts for expense breakdowns.** After checking DealCheck, BiggerPockets, Stessa, Mashvisor, Landlord Studio, Baselane, and Avail — zero use waterfall charts. They all use itemized tables or simple bar charts.
- **The invisible spacer is an accessibility hack.** Screen readers cannot parse the stacked bar with transparent spacers.
- **"Waterfall" in real estate means equity distribution** (LP/GP profit splits), not expense breakdown. Terminology confusion.
- **Abbreviated labels (Mtg, Vac, Ins) compress too much.** On mobile, even these become illegible.

## 2. Where the Waterfall Idea Came From

The waterfall was recommended in `RESEARCH/analysis-page-ux-research.md`, Section C — ADD:

> **Monthly cash flow waterfall** (rent → vacancy → expenses → debt service → net) | Buy & Hold, BRRRR, Creative | Must-have | Replace or supplement break-even chart. Shows WHERE money goes, not just the net number.

This recommendation was generated by AI research, not sourced from any competitor or user study. The research doc's competitive analysis (Section 2) found that **no competitor uses a cash flow waterfall** — DealCheck uses itemized tables with sliders, BiggerPockets uses tables + sliders, Stessa uses dashboard bar charts, and the others all use tabular formats. The waterfall was a generic finance visualization concept applied to RE without precedent.

The implementation plan (`analysis-page-implementation-plan.md`, Feature 6) even flagged the risk:

> "Recharts waterfall via stacked bars is a well-known technique but slightly hacky."

## 6. Recommendation

**Replace the waterfall chart with a proportional breakdown list.**

Reasons:
1. **Zero learning curve.** Every user can read a list with bars. No chart literacy required.
2. **Matches what RE tools actually do.** DealCheck, BiggerPockets, Stessa — all use itemized lists.
3. **Proven in premium finance apps.** Copilot Money uses exactly this pattern in a dark theme.
4. **Better on mobile.** Vertical list scales perfectly; waterfall bars get cramped.
5. **Simpler code.** Pure CSS bars, no Recharts dependency for this component. Easier to maintain.
6. **More accessible.** Screen readers parse lists natively. The invisible spacer hack is eliminated.
7. **Shows amounts AND proportions.** Dollar amounts on the right, bar width shows proportion. Both are visible at once without hovering.

### Component Structure (pseudocode)

```tsx
function CashFlowBreakdown({ scenario }) {
  // Reuse existing data extraction from WaterfallChart
  const { grossRent, deductions, netCashFlow } = extractBreakdown(scenario)
  if (!grossRent) return null

  return (
    <Card title="MONTHLY CASH FLOW BREAKDOWN">
      {/* Gross rent header */}
      <Row label="Gross Rent" amount={grossRent} accent />

      <Divider />

      {/* Each deduction: label, proportion bar, amount, percentage */}
      {deductions.map(d => (
        <Row
          key={d.key}
          label={d.label}
          amount={d.amount}
          pct={(d.amount / grossRent) * 100}
          barWidth={(d.amount / grossRent) * 100}
          barColor="var(--chart-negative)"   // coral for deductions
        />
      ))}

      <Divider />

      {/* Net cash flow footer */}
      <Row
        label="Net Cash Flow"
        amount={netCashFlow}
        pct={(netCashFlow / grossRent) * 100}
        barWidth={(Math.abs(netCashFlow) / grossRent) * 100}
        barColor={netCashFlow >= 0 ? 'var(--chart-accent)' : 'var(--chart-negative)'}
        accent
      />
    </Card>
  )
}

function Row({ label, amount, pct, barWidth, barColor, accent }) {
  return (
    <div className="flex items-center gap-3 py-1.5">
      <span className="w-24 text-xs text-muted shrink-0">{label}</span>
      <div className="flex-1 h-2 bg-layer-1 rounded-full overflow-hidden">
        {barWidth && <div style={{ width: `${barWidth}%`, backgroundColor: barColor }} className="h-full rounded-full" />}
      </div>
      <span className="w-16 text-right text-sm tabular-nums font-medium">${amount}</span>
      {pct != null && <span className="w-12 text-right text-xs text-muted tabular-nums">{pct.toFixed(1)}%</span>}
    </div>
  )
}
```

### What to Reuse from WaterfallChart.tsx

The data extraction logic is already correct:
- Reading `monthly_expense_breakdown` from scenario outputs
- Computing gross rent from `effective_gross_income + vacancy`
- Filtering zero-value categories
- Ordering deductions by `EXPENSE_ORDER`
- Computing net cash flow

Only the rendering layer changes (Recharts BarChart → styled list with CSS bars).
