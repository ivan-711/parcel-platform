# PDF & Print Theme Spec

Definitive specification for Parcel's PDF report generation within the luxury dark
redesign. PDFs remain LIGHT (white background, dark text) for professional printing.
The dark UI produces premium light-page documents -- the same pattern used by Linear,
Notion, and Stripe.

---

## 1. PDF Color Scheme

All colors are defined as `[R, G, B]` tuples for jsPDF's `setTextColor` / `setFillColor`.

```typescript
// --- Brand accent (print-safe) ---
const VIOLET_700: [number, number, number] = [109, 96, 204]   // #6D60CC — headers, rules, section titles
const VIOLET_50:  [number, number, number] = [245, 243, 255]  // #F5F3FF — KPI card fill

// --- Page background ---
const CREAM:      [number, number, number] = [250, 250, 249]  // #FAFAF9 — page bg (not pure white)
const WHITE:      [number, number, number] = [255, 255, 255]  // #FFFFFF — fallback, table alt rows

// --- Text hierarchy (warm stone scale) ---
const CHARCOAL:   [number, number, number] = [12, 11, 10]     // #0C0B0A — property address, hero values
const WARM_800:   [number, number, number] = [41, 37, 36]     // #292524 — body text, table values
const WARM_600:   [number, number, number] = [87, 83, 78]     // #57534E — secondary labels, dates
const WARM_500:   [number, number, number] = [120, 113, 108]  // #78716C — tertiary, metadata, disclaimers
const WARM_300:   [number, number, number] = [214, 211, 209]  // #D6D3D1 — rules, table borders, dividers
const WARM_200:   [number, number, number] = [231, 229, 228]  // #E7E5E4 — alternating row fill

// --- Semantic (cash flow, risk) ---
const SUCCESS_700: [number, number, number] = [4, 120, 87]    // #047857 — positive cash flow
const WARNING_600: [number, number, number] = [217, 119, 6]   // #D97706 — medium risk
const ERROR_700:   [number, number, number] = [185, 28, 28]   // #B91C1C — negative values, high risk
const CRITICAL:    [number, number, number] = [127, 29, 29]   // #7F1D1D — critical risk
```

Rationale: Pure white (#FFFFFF) as page background prints identically to #FAFAF9, but
on-screen PDF previews benefit from the subtle warmth. Stone-scale neutrals replace the
current cool grays (Untitled UI scale) to align with the luxury warm-neutral system.

## 2. Header

Every page receives a consistent header. Replace the filled indigo square + "P" logo
with a refined wordmark and hairline rule.

```typescript
function drawPageHeader(doc: jsPDF): void {
  // Violet accent bar — 2mm tall, full content width
  doc.setFillColor(...VIOLET_700)
  doc.rect(MARGIN, 6, CONTENT_WIDTH, 2, 'F')

  // Wordmark: "PARCEL" in Inter SemiBold (or Helvetica Bold fallback)
  doc.setFont('Inter', 'bold')       // falls back to Helvetica if not embedded
  doc.setFontSize(13)
  doc.setTextColor(...CHARCOAL)
  doc.setCharSpace(1.5)              // generous tracking for luxury feel
  doc.text('PARCEL', MARGIN, 16)
  doc.setCharSpace(0)                // reset

  // Right-aligned report type + date
  doc.setFont('Inter', 'normal')
  doc.setFontSize(7.5)
  doc.setTextColor(...WARM_500)
  doc.text('Deal Intelligence Report', PAGE_WIDTH - MARGIN, 13, { align: 'right' })
  doc.setFontSize(7)
  doc.text(dateStr, PAGE_WIDTH - MARGIN, 17, { align: 'right' })

  // Hairline rule — 0.3pt, warm gray
  doc.setDrawColor(...WARM_300)
  doc.setLineWidth(0.3)
  doc.line(MARGIN, HEADER_HEIGHT, PAGE_WIDTH - MARGIN, HEADER_HEIGHT)
}
```

Key changes from current implementation:
- Violet accent bar replaces the indigo rounded square with "P" text.
- Charcoal wordmark instead of colored brand name -- more authoritative on paper.
- Date pulled into the header for immediate context on every page.
- Tracked letterSpacing (1.5 units) on "PARCEL" for the luxury editorial feel.

## 3. Footer

```typescript
function drawPageFooter(doc: jsPDF, pageNum: number, totalPages: number, dateStr: string): void {
  const y = PAGE_HEIGHT - FOOTER_HEIGHT

  // Top rule
  doc.setDrawColor(...WARM_300)
  doc.setLineWidth(0.2)
  doc.line(MARGIN, y, PAGE_WIDTH - MARGIN, y)

  // Left: branding + timestamp
  doc.setFont('Inter', 'normal')
  doc.setFontSize(7)
  doc.setTextColor(...WARM_500)
  doc.text(`Generated by Parcel  \u00B7  ${dateStr}  \u00B7  parcel-platform.com`, MARGIN, y + 6)

  // Right: page numbers
  doc.text(`${pageNum} / ${totalPages}`, PAGE_WIDTH - MARGIN, y + 6, { align: 'right' })

  // Legal disclaimer — 6pt, extra light
  doc.setFontSize(5.5)
  doc.setTextColor(...WARM_500)
  doc.text(
    'For informational purposes only. Not an appraisal, financial advice, or investment recommendation.',
    MARGIN, y + 10
  )
}
```

Changes: Use the middle dot (\u00B7) separator instead of pipe for elegance. Reduce
disclaimer font to 5.5pt to further de-emphasize. Page numbers use "1 / 3" format
instead of "Page 1 of 3" for brevity.

## 5. Cash Flow Table

```typescript
const ROW_HEIGHT = 7.5                       // slight increase for readability
const LABEL_X = MARGIN + 3
const VALUE_X = PAGE_WIDTH - MARGIN - 3

// Section title
doc.setFont('Inter', 'bold')

// Alternating rows
for (let i = 0; i < rows.length; i++) {
  const bgColor = i % 2 === 0 ? WARM_200 : WHITE   // #E7E5E4 / #FFFFFF
  doc.setFillColor(...bgColor)
  doc.rect(MARGIN, y - 4.5, CONTENT_WIDTH, ROW_HEIGHT, 'F')

  // Label: warm body text
  doc.setFont('Inter', 'normal')
  doc.setFontSize(8.5)
  doc.setTextColor(...WARM_600)              // #57534E — softer than current GRAY_500
  doc.text(row.label, LABEL_X, y)

  // Value: cash-flow coloring
  doc.setFont('Inter', 'bold')               // Courier Bold fallback
  doc.setFontSize(8.5)
  if (row.isCashFlow && row.rawNumeric > 0) {
    doc.setTextColor(...SUCCESS_700)         // green for positive
  } else if (row.rawNumeric !== undefined && row.rawNumeric < 0) {
    doc.setTextColor(...ERROR_700)           // red for negative
  } else {
    doc.setTextColor(...WARM_800)            // neutral dark
  }
  doc.text(row.value, VALUE_X, y, { align: 'right' })

  y += ROW_HEIGHT
}

// Table bottom border
doc.setDrawColor(...WARM_300)
doc.setLineWidth(0.2)
doc.line(MARGIN, y - 3.5, MARGIN + CONTENT_WIDTH, y - 3.5)
```

Changes from current: WARM_200 replaces GRAY_200 for alternating rows. Warmer neutral
on labels. Consistent use of Inter (or Courier fallback) for values.

## 6. Risk Score

Replace the filled badge with a horizontal bar gauge -- more informative and print-safe
than a circular gauge (circles require canvas rendering that degrades in print).

```typescript
function drawRiskScore(doc: jsPDF, score: number, startY: number): number {
  let y = startY

  // Section label
  doc.setFont('Inter', 'bold')
  doc.setFontSize(10)
  doc.setTextColor(...VIOLET_700)
  doc.text('RISK ASSESSMENT', MARGIN, y)
  y += 8

  // Bar track (full width, 4mm tall, rounded)
  const BAR_W = CONTENT_WIDTH * 0.6          // 60% of content width
  const BAR_H = 4
  const BAR_X = MARGIN
  const BAR_RADIUS = 2

  // Background track
  doc.setFillColor(...WARM_200)
  doc.roundedRect(BAR_X, y, BAR_W, BAR_H, BAR_RADIUS, BAR_RADIUS, 'F')

  // Filled portion
  const fillW = (score / 100) * BAR_W
  const color = getRiskColor(score)
  doc.setFillColor(...color)
  doc.roundedRect(BAR_X, y, Math.max(fillW, BAR_RADIUS * 2), BAR_H, BAR_RADIUS, BAR_RADIUS, 'F')

  // Score number — right of bar
  doc.setFont('Inter', 'bold')
  doc.setFontSize(14)
  doc.setTextColor(...color)
  doc.text(`${score}`, BAR_X + BAR_W + 6, y + 3.5)

  // Risk label — further right
  doc.setFont('Inter', 'normal')
  doc.setFontSize(9)
  doc.setTextColor(...WARM_600)
  doc.text(getRiskLabel(score), BAR_X + BAR_W + 20, y + 3.5)

  return y + 14
}
```

The bar gauge is 100% vector (no canvas/image). It prints crisply at any DPI and
communicates the score proportionally -- a visual improvement over the current
badge-only display.

### Font Embedding Strategy

```typescript
// Embed Inter for Pro subscribers (exact brand match)
// ~80KB per weight, loaded as base64 at build time
import interRegularBase64 from '@/assets/fonts/Inter-Regular.base64'
import interSemiBoldBase64 from '@/assets/fonts/Inter-SemiBold.base64'

function registerFonts(doc: jsPDF): void {
  doc.addFileToVFS('Inter-Regular.ttf', interRegularBase64)
  doc.addFont('Inter-Regular.ttf', 'Inter', 'normal')
  doc.addFileToVFS('Inter-SemiBold.ttf', interSemiBoldBase64)
  doc.addFont('Inter-SemiBold.ttf', 'Inter', 'bold')
  doc.setFont('Inter', 'normal')
}

// Free tier: use Helvetica (built-in, zero cost)
// Courier Bold for tabular financial numbers (built-in)
```

Fallback chain: Inter SemiBold > Helvetica Bold > system sans-serif.
For financial values, Inter Bold is preferred; Courier Bold remains acceptable
as a monospaced fallback that ensures column alignment.

## 9. @media print CSS

For browser-based print (Cmd+P on deal pages), provide overrides that produce
a clean light document from the dark UI.

```css
@media print {
  /* Reset dark background to white */
  body,
  html {
    background: #FAFAF9 !important;
    color: #292524 !important;
    -webkit-print-color-adjust: exact;
    print-color-adjust: exact;
  }

  /* Hide non-print UI elements */
  nav,
  .sidebar,
  .no-print,
  .command-palette,
  .toast-container,
  [data-radix-popper-content-wrapper] {
    display: none !important;
  }

  .print-only {
    display: block !important;
  }

  /* Strip shadows and glows */
  * {
    box-shadow: none !important;
    text-shadow: none !important;
  }

  /* Force light surfaces */
  .card,
  [class*="bg-[#0C0B0A]"],
  [class*="bg-[#151413]"],
  [class*="bg-[#1A1918]"] {
    background: white !important;
    border-color: #D6D3D1 !important;
  }

  /* Ensure text is dark */
  [class*="text-[#F0EDE8]"],
  [class*="text-[#D6D3D1]"] {
    color: #292524 !important;
  }

  /* Violet accent stays (print-safe) */
  [class*="text-[#8B7AFF]"] {
    color: #6D60CC !important;
  }

  /* Page break control */

  .page-break-after  { page-break-after: always; }

  /* Clean link display */
  a { color: #6D60CC !important; text-decoration: none !important; }

  /* Table optimization */
  table { border-collapse: collapse; }

  }
```

## 10. jsPDF Quality & Configuration

```typescript
// Document initialization
const doc = new jsPDF({
  unit: 'mm',
  format: 'a4',
  compress: true,                   // gzip streams — reduces PDF size ~40%
  putOnlyUsedFonts: true,           // strip unused font subsets
})

// PDF metadata — makes reports searchable in document management systems
doc.setProperties({
  title: `Deal Analysis — ${deal.address}`,
  subject: 'Real Estate Deal Intelligence Report',
  author: 'Parcel',
  creator: 'Parcel Deal Intelligence Platform',
  keywords: `real estate, deal analysis, ${deal.strategy}`,
})

// Image handling (if charts/logos are ever added as images)
// - Use PNG format, not JPEG, for crisp line art and text
// - Minimum 150 DPI for print (300 DPI preferred)
// - For an 87mm-wide chart: 87mm * 300DPI / 25.4 = 1028px minimum width
// - Add images via: doc.addImage(dataUrl, 'PNG', x, y, w, h)

// Vector-first strategy:
// - All shapes (rects, lines, circles) use jsPDF drawing primitives
// - These render as vector in the PDF — infinite resolution, zero quality loss
// - NEVER rasterize UI components to canvas for the PDF
// - The risk bar, KPI cards, and table rows are all vector

// Line quality
// - Hairlines: 0.2pt (borders, table rules)
// - Standard rules: 0.3pt (section dividers)
// - Accent rules: 0.4pt (violet section underlines)
// - Header bar: 2mm filled rect (not a thick line — avoids printer aliasing)

// File output
const filename = deal.address
  .replace(/[^a-z0-9]/gi, '-')
  .replace(/-+/g, '-')
  .toLowerCase()
doc.save(`${filename}-parcel-report.pdf`)

// For preview modal (blob URL):
// const blobUrl = doc.output('bloburl')
```

## CRITICAL DECISIONS

1. **PDFs remain light.** The dark UI generates white-background documents. This is
   non-negotiable -- lenders, brokers, and attorneys expect white-paper reports. Dark
   PDFs waste ink, degrade when scanned/faxed, and read as screenshots rather than
   professional deliverables.

2. **#6D60CC is the print-safe violet.** The screen violet (#8B7AFF) is too light for
   reliable print reproduction. #6D60CC darkened by ~15% ensures the accent reads
   clearly on both inkjet and laser printers without appearing washed out.

3. **Warm stone neutrals replace cool grays.** Every gray in the current PDF (GRAY_200
   through GRAY_900, sourced from Untitled UI) is replaced with the corresponding warm
   stone value. This aligns the PDF's tonal warmth with the luxury dark UI's neutral
   system and eliminates the dissonance of cool-gray paper exports from a warm-neutral
   interface.

4. **#FAFAF9 page background, not pure white.** The difference is invisible on paper
   but perceptible in on-screen PDF preview. It adds warmth to the preview modal and
   prevents the jarring contrast of pure white inside the dark UI shell.

5. **Horizontal bar gauge replaces badge for risk score.** A proportional bar is more
   informative than a filled rectangle with a number. It communicates magnitude at a
   glance and is 100% vector -- no canvas rendering, no resolution concerns.

6. **Inter font embedding is Pro-tier only.** Embedding two weights adds ~160KB to the
   client bundle. Free-tier reports use Helvetica (built-in, zero cost). This creates
   a tangible quality difference between tiers without degrading the free experience.

7. **Margins increase from 15mm to 18mm.** Premium financial reports use generous
   margins. The 3mm increase per side costs minimal content space but significantly
   elevates the document's perceived quality when printed or displayed in a meeting.

8. **Vector-first rendering, always.** Every shape in the PDF (KPI cards, risk bars,
   table fills, rules) uses jsPDF drawing primitives. No canvas rasterization. This
   guarantees crisp output at any zoom level or print DPI. If charts are added in the
   future, they must be drawn as vector paths, not exported as bitmap images.

9. **Filename convention changes.** Output files become `{address}-parcel-report.pdf`
   (adding "parcel" branding). Consecutive hyphens are collapsed. This improves
   discoverability when reports accumulate in a downloads folder.

10. **PDF preview before download.** Generate via `doc.output('bloburl')` and display
    in an iframe inside a dark modal. Users see exactly what they will share before
    committing. Revoke the blob URL on modal close to prevent memory leaks.
